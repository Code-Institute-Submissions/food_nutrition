About this App

This app is part of my portfolio for my Fullstack Web developer course at Code Institute. It is the project for the Data Centric Development module.

This app allows users to store information about the nutritional value of various food products. I chose to store the values per 100g (or 100ml). Products can be compared and filtered based on various parameters in the dashboard. (See features below.)

This app allows the user to fullfill the following tasks:

Categorize the foods according to food classes.
The App allows the user to create, view, edit and delete classes.
  • A class can only be deleted if it is not being used by any food item.
  • Editing a class will change the class name in all food items that are classified under that class.
  • There is also the functionality to see all the classes. Here, there is also an indication of how many food items make use of each class.
  • When adding a new class or editing an exisiting one, the app will check that that class name does not already exist.

There are three places that will take the user to the page where one can add a new class. The app keeps track of the calling place and will return there on cancelling the process or creating a new class.

Food Items can be viewed, added, edited and deleted.

Each food item will belong to a class. It is possible for the user to create a new food item. The form will have the classes prepopulated. The user will have to select one. There is the option to create a new class with checks on whether it already exists.

There are two ways of viewing the food items.

The accordion view

Here each item is in an accordion panel. I chose to display the info on a card which can rotate to display extra information at the back of the card. Here, the user will also find buttons to edit or delete that particular food item.

Apart from the button saying: "More info and Controls at the back", the curved arrow at the top right corner of the card is a visual indication pointing to the back of the card.

If the user clicks on the delete button, a confirmation window will open. The user can confirm or reject the deletion. If the user proceeds to delete an item, the count will be reduced for the class used by that item. If a user selects to edit the food item, a window will open which will be prepopulated with the information of that item. The user can do the changes necessary and then click the "Update Food Item" button.

If the user chooses to reclassify a food item, then the necessary changes to the quantities of the classes involved will be applied, (increase the new class, decrease the old class). The user can cancel the editing process and will be taken back to the accordion view.

Dashboard

In the dashboard view, the user is able to filter the food items based on the class they belong to, as well as a number of parameter combinations. At the bottom of the page there is a table which initially will show all products, currently up to 1000, but the number of food items shown will be reduced depending on the filters applied.

The name of the food item in the table is a button. Clicking a food item will open a modal which will display the data related to that food item.

I experimented with styling the modal to make it appear as if it is a piece of paper on a desk. As soon as the user hovers over the modal, the paper changes as if pressure or weight has been applied to it. I tried to achieve an organic effect.

The user can close the modal in a number of ways.

  • Clicking on the close button.
  • Clicking on the X at the top right corner of the modal.
  • Clicking on the desk (the white border) around the modal.
  • Pressing the esc button on the computer keyboard.
The contact Form

The form is validated and it also has the appropriate try except blocks to catch errors in the case that the email address is not accepted by the server. In such case, the user is returned to an error page.

Examples of email addresses that will trigger such a response are try@example.dd or another@example

Both these cases pass the validation test but are refused by the email server.
