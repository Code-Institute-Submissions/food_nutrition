About this App

This app is part of my portfolio for my Fullstack Web developer course at Code Institute. It is the project for the Data Centric Development module.

This app allows users to store infromation about the nutritional value of various food products. I chose to store the values per 100g (or 100ml). Users should be aware that this value on its own is not enough as some items which for the sake of this example, might be high in salt might be used in small quantities, while others which are low in salt might be used in larger quantitities. In any case products can be compared and filtered based on various parameters in the dashboard. (See features below.)

This app allows the user to fullfill the following tasks:

Categorize the foods according to food classes.
The App allows the user to create, edit and delete classes.
A class can only be deleted if it is not being used by any food item.
Editing a class will change the name in all food items that are classified under that class.
There is also the functionality to see all the classes. Here there is also an indication of how many food items make use of each class.
When adding a new class, the app will check that that class does not already exist.
Food Items can be viewed, added, edited and deleted.

There are two ways of viewing the food items.

The accordion view

Here each item is in an accordion panel. I chose to display the info on a card which can rotate and show extra information at the back of the card. Here the user will also find buttons to edit or delete that particular food item.

Apart from the button saying:"More info and Controls at the back", the curved arrow at the top right corner of the card is a visual indication pointing to the back of the card.

If the user clicks on the delete button, a confirmation window will open. The user can confirm or reject the deletion. An item will be reduced from the class used by that item. If a user selects to edit the food item, a window will open which will be prepopulated with the information of that item. The user can do the changes necessary and then click the "Update Food Item" button.

If the user chooses to reclassify a food item, then the necessary changes to the quantities of the classes involved will be applied, (increase to the new class, decrease the old class).

The user can cancel the editing process and will be taken back to the accordion view.

There are three places that will take you to the route which allows for the adding a new class. The app keeps track of the calling place and will return there on cancelling or creating a new class. Need to check this. Editing a food item and trying to create a new class from here will return to the add food item.

It is possible for the user to create a new food item. The form will have the classes prepopulated. The user will have to select one. There is the option to create a new class with checks on whether it already exists.

Dashboard

In the dashboard view, the user is able to filter based on classes, and a number of paramter combinations. At the bottom of the page there is a table which initially will show all products, currently up to 1000, but will be reduced depending on the filters applied. The name of the food item in the table is a button.

Clicking a food item will open a modal which will display the data related to that food item.

I experimented with styling the modal to make it appear as if it is a piece of paper on a desk. As soon as the user hovers over the modal, the paper changes as if pressure or weight has been applied to it. I tried to achieve an organic effect.

The user can close the modal in a number of ways.

  • Clicking on the close button.
  • Clicking on the X at the top right corner of the modal.
  • Clicking on the desk (the white border) around the modal.
  • Pressing the esc button on the computer keyboard.

The form is validated and it also has a try except to catch if the email address is not accepted by the server. In this case the user is returned to an error page.
